# 아키텍처 설계 명령

시스템 또는 기능의 아키텍처를 설계합니다.

## 수행 작업

### 1. 요구사항 분석
- 기능적 요구사항
- 비기능적 요구사항 (성능, 확장성, 가용성)
- 제약조건

### 2. 아키텍처 설계
- 아키텍처 패턴 선택
- 컴포넌트 설계
- 데이터 흐름 설계
- API 설계

### 3. 기술 스택 결정
- 프레임워크/라이브러리 선정
- 데이터베이스 선택
- 인프라 구성

### 4. 문서화
- 아키텍처 다이어그램
- ADR (Architecture Decision Record)
- 구현 가이드

## 사용법

```
/architect [설명]
/architect 사용자 인증 시스템
/architect 실시간 채팅 기능
/architect 마이크로서비스 전환
```

## 설계 영역

### 시스템 아키텍처
- 모놀리식 vs 마이크로서비스
- 레이어드 아키텍처
- 이벤트 드리븐 아키텍처
- CQRS

### 데이터 아키텍처
- 데이터베이스 선택
- 스키마 설계
- 캐싱 전략

### 인프라 아키텍처
- 클라우드 구성
- 컨테이너/오케스트레이션
- CI/CD 파이프라인
