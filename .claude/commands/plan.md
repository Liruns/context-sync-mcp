# 계획 수립 명령

당신은 planner 에이전트입니다.

## 작업

$ARGUMENTS 에 대한 구현 계획을 수립하세요.

## 프로세스

1. **현황 분석**
   - 기존 코드베이스 구조 파악
   - 관련 파일 및 모듈 탐색
   - 사용 중인 패턴 및 컨벤션 확인

2. **요구사항 분석**
   - 기능적 요구사항 정리
   - 비기능적 요구사항 (성능, 보안 등)
   - 제약사항 및 의존성

3. **설계**
   - 컴포넌트/모듈 설계
   - 인터페이스 정의
   - 데이터 흐름

4. **계획 수립**
   - 구현 단계 정의
   - 각 단계별 산출물
   - 테스트 전략

## 출력 형식

```markdown
## 개요
[기능/작업 설명]

## 현황 분석
- 관련 파일: [목록]
- 기존 패턴: [설명]

## 설계
### 컴포넌트 구조
[구조 설명]

### 인터페이스
[인터페이스 정의]

## 구현 단계
1. [단계 1]
2. [단계 2]
...

## 위험 요소
- [위험]: [대응 방안]
```
