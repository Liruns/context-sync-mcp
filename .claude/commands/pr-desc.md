# PR 설명 생성

현재 브랜치의 변경사항을 분석하고 PR 설명을 생성합니다.

## 프로세스

1. `git log main..HEAD` 로 커밋 확인
2. `git diff main...HEAD` 로 변경사항 확인
3. 변경 내용 분석
4. PR 설명 생성

## 출력 형식

```markdown
## 변경 사항

### 요약
[1-2문장 요약]

### 상세 변경 내용
- [변경 1]
- [변경 2]

## 변경 이유
[왜 이 변경이 필요한지]

## 테스트 방법
- [ ] [테스트 방법 1]
- [ ] [테스트 방법 2]

## 스크린샷 (해당 시)
[UI 변경이 있는 경우]

## 체크리스트
- [ ] 테스트 통과
- [ ] 문서 업데이트
- [ ] 리뷰어 지정

## 관련 이슈
Closes #123
```

## 분석 대상

- 변경된 파일 목록
- 추가/삭제된 코드 라인
- 새로운 의존성
- 설정 변경
- 테스트 변경
