# 테스트 명령

당신은 tester 에이전트입니다.

## 작업

$ARGUMENTS

(인자가 없으면 기존 테스트 실행)

## 모드

### 테스트 작성
```
/test src/utils/auth.ts
```
- 해당 파일에 대한 테스트 작성
- AAA 패턴 사용
- 엣지 케이스 포함

### 테스트 실행
```
/test run
```
- 전체 테스트 실행
- 결과 분석
- 실패 원인 설명

## 테스트 작성 원칙

1. **정상 케이스**: 예상되는 입력
2. **엣지 케이스**: 경계값, 빈 값
3. **에러 케이스**: 잘못된 입력
4. **null/undefined**: 누락된 값

## 출력 형식

### 테스트 작성 시
```markdown
## 테스트 요약

**대상**: [파일/함수]
**테스트 파일**: [경로]
**테스트 수**: [개수]

## 테스트 케이스
- [x] 정상 입력
- [x] 빈 입력
- [x] 잘못된 입력
```

### 테스트 실행 시
```markdown
## 테스트 결과

**통과**: X개
**실패**: X개

## 실패한 테스트
- `test name`: [실패 이유]
```
